MODEL OPTIMIZATION / EDGE DEVICE

RetinaFace Optimization

경량화 시 발생하는 성능 저하를 최소화하기 위해 아키텍처 구조를 심층 분석하여 문제를 정의하고, 성능 개선을 위해 FPN 재설계, Attention 기법 도입, Channel Shuffle 및 Deformable Convolution을 적용하여 최적화를 수행했습니다.

Problem Definition

RetinaFace는 RetinaNet 기반 구조를 가진 모델임을 파악 후, RetinaNet의 문제점을 분석했습니다. 그 결과, Detection Head 부분에 많은 연산 부담이 이루어지며 경량화 백본(MobileNet v1) 도입시 성능이 13% 급감합니다.

  • Edge Device에서의 실시간 구동 제약 (낮은 FPS).
  • 작은 객체(Small Face)에 대한 탐지 정확도 유지의 기술적 한계.
  • 모델 경량화 시 발생하는 급격한 성능 저하 문제.

Methodology: Developed Architecture

1. ECA-CBAM

중요한 채널/공간 정보를 강조하여 Head 입력을 정제합니다. CBAM의 MLP를 1D Conv(ECA)로 대체해 연산 부담을 줄이면서 표현력을 유지할 수 있도록 합니다.

2. WFPN (Weighted Feature Pyramid Network)

FPN의 단순 sum 대신 학습 가능한 가중치 결합을 도입해, 스케일별 중요도를 학습하며 정보 손실을 줄이고 성능을 향상시킵니다.

3. SCM (Shuffle Context Module)

채널 수를 줄여 연산을 억제한 뒤, 채널 셔플로 컨텍스트를 섞어 효율적으로 표현력을 강화합니다.

4. Deformable Convolution

offset 기반 sampling으로 소형/변형 얼굴에서도 안정적으로 특징을 포착해 탐지 성능을 보완합니다.
